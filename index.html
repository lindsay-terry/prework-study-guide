HTML

  • The head element contains information about the webpage.
  • The body element represents the visibile content shown to the user.
  • Use semantic elements whenever possible.
  • Element refers to entire element, from opening to closing tag.
  • Tag only refers to what appears inside angle brackets.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Connect external CSS stylesheets to an html document with a link tag.
  • Declaration blocks are group in left opening and right closing curly brackets.
  • Declaration blocks can be nested.
  • Declarations inside blocks must be separated with a semicolon.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • Don't make changes in the main branch! Check what branch you are working in before making changes.
  • git add -A: Add the changed files in our working directory to the staging area of Git.
  • git commit -m "descriptive comment": Commit staged changes to repository with a description of what was done.
  • git pull origin main: Verify main branch is up to date before opening a pull request.
  • git push origin branchname: Push up all changes made to working branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Semicolon at the end of a line indicates where the statement ends.
  • For loops perform tasks multiple times.
  • Arrays store related data items in a single variable.
  • To execute a function, you must call it in your code.
  • Variables must be written as defined to pull the correct information.
